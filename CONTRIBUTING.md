# Contributing to Kinnook

Thanks for your interest in Kinnook — a self-hosted family hub. This guide covers
how to get set up, run the checks, and open a good PR. Keep it practical; when in
doubt, match the surrounding code.

## Local setup

Kinnook runs as a docker compose stack driven by a root `./nook` bash CLI.

```bash
git clone <repo-url> nook
cd nook
./nook up          # first run bootstraps infra/compose/.env with generated secrets
```

`./nook up` brings up postgres, the API, PowerSync, and Caddy. For web
development with hot reload:

```bash
./nook web         # Vite HMR for apps/web
```

Other useful CLI verbs: `./nook down`, `rebuild`, `migrate`, `backup`,
`restore`, `doctor`, `admin`, `logs`, `status`.

Requires Docker and Node 20.

## Project layout

- `apps/api` — backend (TypeScript, lambda-api, bundled with esbuild; Postgres
  migrations via node-pg-migrate). Has its own `package.json` / `package-lock.json`.
- `apps/web` — kiosk / web app (React + Vite). Its own `package.json` too.
- `apps/ios` — SwiftUI app (Xcode project generated by XcodeGen from
  `project.yml` — run `xcodegen generate` after adding files).
- `infra/compose` — docker compose stack (postgres, api, powersync, caddy,
  backup, optional observability).
- `docs/` — project documentation.

This is **not** an npm-workspaces monorepo: `apps/api` and `apps/web` each manage
their own dependencies with npm.

## Tests & typecheck

Run these in the relevant app before opening a PR. CI runs all of it.

```bash
# API — vitest + Testcontainers integration (needs Docker running)
cd apps/api && npm test
cd apps/api && npm run typecheck

# Web — vitest / jsdom
cd apps/web && npm test
cd apps/web && npm run typecheck
```

## Commits

Use conventional-commits-style messages with a scope:

```
feat(meals): add recipe scheduling
fix(chores): correct approval ordering
docs: expand contributing guide
ci: cache node_modules
```

## Pull requests

- **Keep PRs focused** — one logical change per PR is much easier to review.
- **CI must pass** (tests + typecheck for both apps). See
  `.github/workflows/ci.yml`.
- **Add tests** for behavior changes — API changes get integration tests, web
  changes get component/unit tests.
- **Update `CHANGELOG.md`** — add an entry under the `[Unreleased]` section.
- Update relevant docs when you change behavior or config.

## Questions

Open a **GitHub Issue** for bugs and feature requests, or start a **GitHub
Discussion** for questions and ideas. For security issues, see
[`SECURITY.md`](SECURITY.md) — please report those privately, not as public
issues.

By contributing, you agree to abide by our
[Code of Conduct](CODE_OF_CONDUCT.md).
